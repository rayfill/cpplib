C++による自作のライブラリです。
一部のOSやデバイスなど外部システムに直接つながるクラスを除いてユニットテストが実装されています。
基本的にヘッダファイル内にすべてを詰め込んだtemplateクラスで構成されています。
またできる限りC++規格上に収まるように作っています。Endian関係などビットレベルで直接扱わないとダメな部分は無理ですが、それ以外であればreinterpret_castをつかっている部分以外はかなりの移植性を持たせたつもりです（実際Makeファイルでは大体のテストがpedanticオプションでコンパイルされるようにしています）。
MPI(多倍長整数)クラスやDIBクラス、Blitterクラスなどはテンプレートのインライン展開および最適化を期待して作られているため、最適化レベルを多少は上げて作成しないととんでもなく遅くなることがあります。
ただしインライン展開および最適化されればクラスレベルで抽象化している部分がすべて畳み込まれるため、実行速度はアセンブラ直書きに迫るところまでいけるものもあります。

実際のところ、一番時間がかかってるのはCryptography(およびその技術的基底となっている多倍長整数クラス)関連でしょうか。
暗号関連をほとんど一から覚えながら、最適化方法探して実装していったものです。
とりあえずJava実装の数倍程度の速度が出るところまではこぎつけたのですがGNU bnライブラリ（GNUの多倍長整数Cライブラリ）には結構水をあけられています・・・
やっぱりRVO(戻り値最適化)仕切れていない部分のコストとか内部表現を持っているSTL vectorのコストを削らないと追いつけそうに無いですね・・・

各ライブラリの使い方は一部のものに関してはDoxygen用の記述がされているものもありますが、何も書いて無いものの方が多数を占めています(もともと誰かに見せる、とかを前提にしていなかったもので・・・)。

・各ディレクトリ以下にあるものの簡単な説明。
├─Cryptography/ 暗号関連です。AES(AdvancedEncryptionStandard)とRSAの実装があります。ほかに暗号用乱数のインタフェースもあります。RSA-PCKS1に準拠して無いので相互運用性はありません。
│  └─Mode/ 暗号フィードバックモードのCBCの実装があります。
├─DateTime/ ユリウス日換算の日付クラスです。中身の実装はまだほとんどされていません。
├─Graphics/ Windows用2Dグラフィックスクラス群です。位置、ピック、描画、Ｚオーダがあります。
├─Image/ RGB<->YCbCr色変換に関するクラス群です。
├─IO/ 入出力クラス群です。ビット単位のストリーム、メモリへのストリーム、エンディアンコンバータなどがあります。
├─math/ 数学クラス群です。DCT(作りかけ)、多倍長整数、行列、メルセンヌツイスタ、有理数、エラトステネスの篩、幾何要素などがあります。
│  └─theory/ 整数論、代数論のアルゴリズムです。
├─net/ ネットワークサービス用の抽象基底のみが用意されています。(実際にはこの上にHTTPやFTPアクセスサービスを構築する予定です)
├─PosixThread/ ThreadのPosix版実装です。作ってる最中にCygwinのPthread実装がまともじゃないことがわかったので放置中・・・
├─Socket/ ネットワークソケットクラスです。クライアントソケット、サーバソケット、InternetProtocolなどのクラスがあります。
├─support/ サポート用クラス群が入る予定です。現状ではVisual C++.Netのメモリアロケータが死ぬほど遅いのを何とかするためのアロケータのみがあります(MPI(多倍長整数)クラスで使用)。
├─tests/ 各種クラスたちのユニットテスト群が入っています。CppUnitを使うことでテストできます。
├─text/ 文字列関連クラス群があります。レキシカルキャスト（atoiなどのジェネリック版）と文字種コンバータ(MutiByte<->WideCharacters)があります。
│  ├─parser/ 言語解析用のヘルパ・・・なんですが、イマイチ設計が気に食わないので放置されてます。
│  ├─regex/ 正規表現クラス。トークン解析して状態機械を作るところまではできています。あとは動かすだけであればマッチルーチン。実用性を考えるのであれば、その後の操作系(置換とか)やDFAへの変換サポートなどを作る必要があります。
│  ├─transcode/ UTF-16<->UTF-8変換器があります。
│  └─xml/ XMLパーサがあります。ただしW3Cの規格には沿って作ったわけではないので簡易な解析しかできません。代わりに簡易なXPathをサポートしています（DOM Level3サポートすれば本家規格でもXPathつきますが・・・）。
├─Thread/ スレッドサポートです。スレッドグループ、回収可能スレッドグループなどがあります。ベーススレッドの実体はWinThreadかPosixThread以下です。実用にはあとはスレッドプールも作る必要がありそうですね。
├─Time/ 時刻クラス・・・ですが、実体はWinTime/以下でしかもガワしか作ってません。
├─util/ 各種ユーティリティクラスがあります。デザインパターン物クラスなど。CalculateとかRangeなどはmath以下に移したほうがいいかも・・・
│  ├─Compression/ 圧縮クラスです。HuffmanとLZSSがありますがHuffmanしか実装されていません。
│  ├─format/ Exifデータフォーマットサポートクラスです。
│  └─hash/ ハッシュアルゴリズムです。SHA1とMD5がありますが、SHA1しか実装していません。
├─Win32/ Win32依存クラス群です。DIB、デバイスコンテキスト、無効リージョン、ファイル、ファイル検索、ファイルストリーム(中途半端)、マウス(中途半端)、ノイズ(セキュリティ用途向け)、WindowsWaveファイルがあります。
├─WinThread/ Win32スレッドクラス群です。クリティカルセクション、イベント、ミューテックス、スレッドベースがあります。
└─WinTime/ 時刻クラス実体。ただし未実装です。

基本的にCygwin上でg++ 3.4.4 (Cygming special) とCppUnit 1.10.2をつかってテストしています。
大体のものに関してはVisualStudio.Net2003上でもビルドなどはした経験があるのでそのままincludeして使うことができます。
Cryptography, Thread(WinThread含む), math, Graphics, Win32などは動作することを確認してあります。
